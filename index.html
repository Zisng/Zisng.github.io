Spark-Pyspark

Spark-Pyspark基础编码环境

本机pyspark环境配置

  • 安装anaconda环境
  • 配置pycharm本地python解释器
  • 运行本地代码
    from pyspark import SparkConf, SparkContext
    import os
    os.environ[‘PYSPARK_PYTHON’]=‘D:\Anaconda\anaconda\envs\pyspark\python.exe’
    if name == “main”:
    conf = SparkConf().setAppName(“WordCountHelloWorld”).setMaster(“local[*]”)
    sc = SparkContext(conf=conf)
    file_rdd = sc.textFile(“…/data/input/words.txt”)
    words_rdd = file_rdd.flatMap(lambda line: line.split(" "))
    words_with_one_rdd = words_rdd.map(lambda x: (x, 1))
    result_rdd = words_with_one_rdd.reduceByKey(lambda a, b: a + b)
    print(result_rdd.collect())

远程SSH Linux链接

在本地运行，可能会受到一些限制，而且在处理大量数据时，这些限制可能会影响pyspark运行效果，导致一系列性能问题。所以采用远程链接的方式来处理大大规模数据，用ssh远程方式运行pyspark。

  • 配置远程ssh linux解释器

    • 链接设置里的SSH Interpreter
    • 点击下图中的ok
    • 输入密码
    • 选择远程链接的解析器位置
      /export/server/anaconda3/envs/pyspark/bin/python3.8

  • 查看虚拟机上是否同步了pycharm中的文件

  • 运行ssh代码文件
    from pyspark import SparkConf, SparkContext
    import os
    os.environ[‘PYSPARK_PYTHON’]=‘/export/server/anaconda3/envs/pyspark/bin/python3.8’
    if name == ‘main’:
    conf = SparkConf().setAppName(“WordCountHelloWorld”).setMaster(“local[*]”)
    sc = SparkContext(conf=conf)
    file_rdd = sc.textFile(“file:///tmp/pycharm_project_96/data/input/words.txt”)
    words_rdd = file_rdd.flatMap(lambda line: line.split(" "))
    words_with_one_rdd = words_rdd.map(lambda x: (x, 1))
    result_rdd = words_with_one_rdd.reduceByKey(lambda a, b: a + b)
    print(result_rdd.collect())

Spark-Local

Spark本地全流程配置及操作

基础环境

  • 环境变量

    • 配置Spark有如下5个环境变量需要设置

    #JAVA_HOME

    export JAVA_HOME=/export/server/jdk

    export PATH=$PATH:$JAVA_HOME/bin

    export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

    #HADOOP_HOME

    export HADOOP_HOME=/export/server/hadoop

    export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

    #HADOOP_CONF_DIR

    export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop

    #PYSPARK_PYTHON

    export PYSPARK_PYTHON=/export/server/anaconda3/envs/pyspark/bin/python

    #SPARK_HOME

    export SPARK_HOME=/export/server/spark

  • 安装anaconda

    • 把资源包Anaconda3-2021.05-Linux-x86_64.sh文件放到文件夹下
    • 运行Anaconda3-2021.05-Linux-x86_64.sh

    sh ./Anaconda3-2021.05-Linux-x86_64.sh

    • 输入安装路径

    /export/server/anaconda3

    • 安装完成之后重启虚拟机

    看到base（默认的虚拟环境）开头就安装好了

  • 创建虚拟环境

    • 创建虚拟环境 pyspark, 基于Python 3.8

    conda activate base

    conda create -n pyspark python=3.8

    • 切换到虚拟环境内

    conda activate pyspark

    • 在虚拟环境内安装包

    pip install pyhive pyspark jieba -i https://pypi.tuna.tsinghua.edu.cn/simple

安装spark

  • 解压spark包

tar zxvf spark-3.2.0-bin-hadoop3.2.tgz

  • 设置软链接

ln -s spark-3.2.0-bin-hadoop3.2/ spark

  • 测试

./pyspark

  • 在浏览器内打开网页

Docker

docker全流程配置及应用

一、docker初始环境

  • 虚拟机需要链接网络，安装yum工具
    yum install -y yum-utils device-mapper-persistent-data lvm2 --skip-broken

安装完成

二、配置网卡转发

  • 写入
    cat < /etc/sysctl.d/docker.conf
    net.bridge.bridge-nf-call-ip6tables = 1
    net.bridge.bridge-nf-call-iptables = 1
    net.ipv4.conf.default.rp_filter = 0
    net.ipv4.conf.all.rp_filter = 0
    net.ipv4.ip_forward=1
    EOF
  • 重新加载内核参数
    modprobe br_netfilter
    sysctl -p /etc/sysctl.d/docker.conf

三、利用域名进行docker安装

  • 用阿里云提供的docker专属repo仓库
    curl -o /etc/yum.repos.d/docker-ce.repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
  • 更新yum缓存
    yum clean all && yum makecache

更新缓存成功

  • 查看源中可用版本
    yum list docker-ce --showduplicates | sort -r
  • yum安装
  • 查看docker版本，验证是否成功
    docker -v
  • 配置镜像加速器
    mkdir -p /etc/docker

touch /etc/docker/daemon.json

vim /etc/docker/daemon.json
加入
{
“registry-mirrors” : [
“https://8xpk5wnt.mirror.aliyuncs.com”
]
}

  • 启动docker
    • 一定要关闭防火墙！！！
      systemctl stop firewalld
- docker配置文件重新加载
systemctl daemon-reload

- 设置开启自启动
systemctl enable docker

- 启动docker服务
systemctl start docker

  • 查看docker信息

    • 查看 Docker 守护进程的详细信息
      docker info
    • 列出当前正在运行的容器
      docker ps
    • 列出当前系统中所有可用的 Docker 镜像
      docker images
    • 查找并返回指定的命令在系统中的安装路径
      which docker
    • 列出当前所有进程，并在输出中过滤包含关键字 docker的进程信息
      ps aux |grep docker
    • 查找、检查 containerd 服务的运行状态
      ps aux|grep containerd
      systemctl status containerd

四、用docker运行nginx

  • 搜索一下远程仓库中的镜像文件是否存在
    docker search nginx
  • 拉取，下载镜像
    docker pull nginx
  • 查看镜像
    docker images
  • 运行镜像
    运行出具体内容，在容器中就跑着一个nginx服务
    docker run -d -p 80:80 nginx
  • 查看容器是否在运行
    docker ps
  • 访问网站
    192.168.88.161:80
  • 停止容器
    docker stop 17e4e6b1f7ab
  • 恢复容器
    docker start 17e4e6b1f7ab

五、镜像

  • 获取镜像

    • 查看本地的镜像文件
      docker images
      docker image ls
    • 下载docker镜像
      docker pull centos
    • 查看docker镜像的存储路径
      docker info |grep Root

    具体位置

  • 查看镜像

    • 查看所有镜像
      docker images
    • 查看具体镜像
      docker images nginx
    • 指定tag查看
      docker images centos:7.8.2003
    • 只列出镜像id
      docker images -q
    • 格式化显示镜像
      docker images --format
    • 搜索镜像
      docker search centos

  • 删除镜像

    • 这里下载 体积较小的hello-world镜像进行验证
      docker pull hello-world
    • 删除容器，指定id的前三位即可
      docker rmi feb

  • 镜像管理

    • 批量删除镜像（慎用）
      docker rmi ‘docker images -aq’
    • 批量删除容器
      docker rm ‘docker ps -aq’
    • 导出镜像
      docker image save centos:7.8.2003 > /export/software/centos1.8.2003.tgz
    • 导入镜像
      docker image load -i /export/software/centos1.8.2003.tgz

  • 去DockerHub搜索并拉取一个Redis镜像

    • 去DockerHub搜索Redis镜像
      docker search redis
    • 查看Redis镜像的名称和版本
      docker images | grep redis
    • 利用docker pull命令拉取镜像
      docker pull redis:latest
    • 利用docker save命令将 redis:latest打包为一个redis.tar包
      docker save -o redis.tar redis:latest
    • 利用docker rmi 删除本地的redis:latest
      docker rmi redis:latest
    • 利用docker load 重新加载 redis.tar文件
      docker load -i redis.tar

六、容器

  • 运行容器，且进入容器内，且在容器内执行某个命令
    docker run -it centos:7.8.2003 sh
  • 开启一个容器，让它帮你运行某个程序
    docker run centos:7.8.2003 ping baidu.com
  • 运行一个活着的容器，docker -ps可以看到的容器
    docker run -d centos:7.8.2003 ping baidu.com
  • 丰富docker运行的参数
  • 查看容器日志的玩法，刷新日志
    docker logs -f bbdd89dc5287
  • 进入正在运行的容器空间内
    docker exec -it bbdd89dc5287 bash
  • 查看容器的详细信息，用于高级的调试
    docker container inspect bbdd89dc5287
  • 容器的端口映射
    docker run -d --name bigdata_nginx -p 85:80 nginx
- 查看容器的端口转发情况

七、创建并运行一个容器（案例）

  • 创建并运行进入容器
    docker run -it --name my_container5 nginx /bin/bash
  • 进入目录
    cd /usr/share/nginx/html/
  • 修改文件
    sed -i -e ‘s#Welcome to nginx#fsaf#g’ -e ‘s###g’ index.html
  • 退出
    exit

八、数据卷

  • 创建和查看数据卷
    • 创建数据卷
      docker volume create html
    • 查看所有数据
      docker volume ls
    • 查看数据卷详细信息卷
      docker volume inspect html
    可看到，我们创建的html这个数据卷关联的宿主机目录是/var/lib/docker/volumes/html/_data
  • 挂载数据卷
    • 挂载一个数据卷到某个容器内目录
      docker run -it --name bigdata -v /export/data/docker-data/centos/:/root/data_container centos:7.8.2003
      cd /root/data_container
      touch 1.txt
    到目录下查看
    • 配置数据卷容器
      创建启动c3数据卷容器：docker run -it --name=c3 -v /volume centos:7
    创建启动c1 c2 容器
    docker run -it --name=c1 --volumes-from c3 centos:7 docker run -it --name=c2 --volumes-from c3 centos:7
  • 给nginx挂载数据卷
    • 创建容器并挂载数据卷到容器内的HTML目录
      docker run --name ln -v /export/data/docker-data/nginx-html/:/usr/share/nginx/html -p 80:80 -d nginx
    • 进入html数据卷所在位置，并修改HTML内容

9.docker应用部署

  • 部署mysql
    • 搜索mysql镜像
      docker search mysql
    • 拉取mysql镜像
    • 创建容器，设置端口映射、目录映射
      在/export/data/docker-data/目录下创建mysql目录用于存储mysql数据信息
      mkdir -p /export/data/docker-data/mysql
      cd /export/data/docker-data/mysql
      docker run -id -p 3306:3306 --name=bigdata_mysql -v $PWD/conf:/etc/mysql/conf.d -v $PWD/logs:/logs -v $PWD/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=hadoop mysql:5.6
    • 进入容器，操作mysql
      docker exec –it bigdata_mysql /bin/bash
  • 部署Tomcat
    • 搜索tomcat镜像
      docker search tomcat
    • 拉取tomcat镜像
      docker pull tomcat
    • 创建容器，设置端口映射、目录映射
      创建目录tomcat，并进入目录下
    docker run -id --name=c_tomcat -p 8080:8080 -v $PWD:/usr/local/tomcat/webapps tomcat
    • 使用外部机器访问tomcat
      （但是当前资源未公开，不过配置成了）
  • 部署Redis
    • 搜索redis镜像
      docker search redis
    • 拉取redis镜像
      docker pull redis:5.0
    • 创建容器，设置端口映射
      docker run -id --name=c_redis -p 6379:6379 redis:5.0
